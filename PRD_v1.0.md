## 项目背景与目标

### 背景

传统招标文件编制存在条款复用率低、范本版本管理薄弱、跨角色协作成本高、历史归档内容易受后续修改影响等问题。项目分期建设：**本期**聚焦品类、资源与范本编制闭环；**由范本生成招标文件（变量定稿、成品导出）暂不实施**，纳入后续迭代。

### 目标

- 建立可复用的资源库与范本库，提升编制效率。
- 形成资源到范本的编制闭环（含导入、在线编辑、预览、下载、发布与归档）；延伸至招标文件的全链路留待后续版本。
- 支持导入、在线编辑、发布、归档与导出。
- 保障归档版本稳定，避免因引用资源变化导致历史内容漂移。
- 建立操作审计与关键状态追踪能力。

## 页面层级功能与操作逻辑

本文档从此处开始按“页面层级”组织需求，每个页面均由“综述 + 业务流程”组成，统一描述粒度并直接对应实现与测试。

### 数据看板

#### 综述

数据看板用于展示系统运行总览，包含品类、范本、资源等核心统计及待办信息，为运营与编制人员提供入口导航与健康度判断。

#### 业务流程

1. 用户进入页面，系统读取本地业务数据。
2. 系统计算总量、状态分布与待办数据。
3. 页面渲染指标卡片、待办项与快捷操作入口。
4. 用户切换菜单后返回，统计结果按最新数据刷新。

### 品类管理

#### 综述

品类管理负责维护“产业领域 -> 采购分类 -> 品类”三级结构，是资源与范本归属的基础主数据模块，提供同步与日志追溯能力。

#### 业务流程

1. 用户点击左侧菜单【品类管理】进入页面，系统加载三级树与统计卡。
2. 用户点击产业领域/采购分类前的展开图标，查看下级节点。
3. 用户在顶部搜索框输入关键字，系统实时过滤匹配节点。
4. 用户点击右上角【同步信息】按钮，系统进入加载态并调用同步接口更新数据。
5. 用户点击右上角【操作日志】按钮，系统弹窗展示同步日志并支持分页切换。

### 资源管理（一级分组）

#### 综述

资源管理用于沉淀可复用条款内容，统一支持草稿、发布、绑定、删除、日志、版本快照，支撑后续范本编制复用。

#### 业务流程

1. 用户点击左侧菜单【资源管理】并选择子菜单（文本/资格条件/评标办法/合同条款）进入页面。
2. 系统加载对应模块资源列表，默认显示左侧列表与右侧详情区。
3. 用户点击【新建】或【编辑内容】进入弹窗，执行保存草稿或发布操作。
4. 用户在详情区点击【绑定章节】，在弹窗内勾选范本章节并点击【确认绑定】形成引用关系。
5. 用户点击资源行的【日志】图标，弹窗查看该资源的操作日志与分页记录。

#### 文本管理

##### 综述

文本管理维护通用文本条款，强调章节化复用与版本追踪，是范本正文的主要资源来源。

##### 业务流程

1. 用户点击【资源管理】->【文本管理】进入页面后，点击左上【新建文本】。
2. 在编辑弹窗填写名称和正文后，点击【保存草稿】或【立即发布】。
3. 发布后，用户点击详情区【绑定章节】并在弹窗中选择目标范本章节，点击【确认绑定】保存。
4. 系统将已发布文本同步到关联的非归档范本章节内容。
5. 用户点击列表行【日志】查看历史记录，或点击详情区【删除】执行逻辑删除。

#### 资格条件管理

##### 综述

资格条件管理复用资源管理通用能力，面向资质条款场景，确保资格要求模板化沉淀与可复用。

##### 业务流程

1. 用户点击【资源管理】->【资格条件管理】进入页面，点击【新建资格条件】或【编辑内容】。
2. 在弹窗中修改条款后点击【保存草稿】或【发布】，系统写入版本快照。
3. 用户点击【绑定章节】将资格条款绑定到范本对应章节。
4. 用户通过【日志】图标查看变更记录，必要时点击【删除】执行逻辑删除。

#### 评标办法管理

##### 综述

评标办法管理用于维护评审相关规则文本，支持草稿编制、发布、绑定与追溯，保障评标条款一致性。

##### 业务流程

1. 用户点击【资源管理】->【评标办法管理】进入页面，选择【新建】或【编辑内容】。
2. 在弹窗中录入评标条款，点击【保存草稿】或【发布】。
3. 用户点击【绑定章节】并选择范本章节，点击【确认绑定】实现复用。
4. 用户点击列表【日志】图标查看发布、绑定、删除等操作过程。

#### 合同条款管理

##### 综述

合同条款管理用于沉淀合同类标准文本，保持与其他资源模块一致的维护、发布、绑定、审计流程。

##### 业务流程

1. 用户点击【资源管理】->【合同条款管理】进入页面并点击【新建合同条款】。
2. 在编辑弹窗填写内容后点击【保存草稿】或【发布】。
3. 用户点击【绑定章节】并选择目标范本章节后点击【确认绑定】。
4. 条款不再使用时，用户点击【删除】并确认，系统执行逻辑删除并保留审计。

### 范本管理

#### 综述

范本管理负责范本全生命周期处理，覆盖创建、导入、筛选、在线编辑、预览、下载、发布、删除与日志查询。

#### 业务流程

1. 用户点击左侧菜单【范本管理】进入页面，通过筛选区和搜索框定位目标范本。
2. 用户点击【新建范本】或【导入文档】创建草稿范本。
3. 在列表操作列点击【编辑】进入在线编辑器完善章节内容。
4. 编辑完成后点击【保存并返回】，系统回写范本并更新进度和更新时间。
5. 用户在操作列按需点击【发布】、【预览】、【下载】、【日志】、【删除】。

### 在线编辑（WPS 文档中台）— 页面综述与主流程

#### 综述

在线编辑是范本内容编制核心场景：业务系统通过 **WPS 文档中台** 提供的 WebOffice 编辑能力与 **JSSDK**，完成 DOCX 在线协同编辑；业务侧右侧面板继续承担资源插入、变量插入；文档落盘与版本以中台回调 **「上传文件新版本」** 为准，业务系统将最新 DOCX 解析并回写范本章节树。网络与中台地址、网闸、DNS 及开通流程按《WPS云文档接入文档(2)》执行。

#### 业务流程

1. 用户在范本列表点击【编辑】：业务系统完成鉴权与文件准备后，按《文档中台接口文档》调用 **获取在线编辑链接**（`GET …/open/api/edit/v1/files/{file_id}/link`，`file_id` 与范本文档一一对应），打开 **WebOffice** 编辑页。
2. 文档打开时，文档中台按回调约定请求业务系统 **「获取文件元数据」**（默认 `/v1/3rd/file/info` 等，可在应用管理平台配置），业务系统返回下载地址、权限与水印等；中台加载 DOCX。
3. 用户在右侧系统面板点击【插入】（资源）或【插入变量】（变量库），由业务逻辑写入编辑区（可与 JSSDK 协同）；若插入失败，系统先自动重试，再回退到插入队列或手工粘贴方案。
4. 用户保存或关闭触发中台 **「上传文件新版本」** 回调（默认 `/v1/3rd/file/save`）：业务系统接收 DOCX，更新存储与版本号，并解析回写范本章节、进度与审计信息；用户点击【保存并返回】退出编辑壳层后回到列表。
5. 返回范本列表后，用户可继续点击【预览】、【下载】或【发布】；预览可按需采用中台 **在线预览** 能力或沿用业务侧预览。

## 全局规则（简版）

### 总目标

在统一产品口径下，实现“品类 -> 资源 -> 范本 -> 在线编辑”的稳定闭环，确保页面功能可执行、可追溯、可验收。

### 规则1：状态与流程统一

- 资源状态统一为“草稿、已发布”。
- 范本状态统一为“草稿、已发布、已归档”。
- 所有页面交互需遵循“校验 -> 执行 -> 反馈”标准流程。

### 规则2：一致性与版本稳定

- 已归档范本按快照读取，不受后续资源变更影响。
- 资源更新仅联动非归档范本。
- 绑定关系需去重，禁止向归档范本新增绑定。

**细化说明（资源管理 × 已归档范本）**

- **适用范围**：资源管理下各子模块（文本、资格条件、评标办法、合同条款）维护、发布条款及与范本章节的绑定与联动，均须遵守本条。
- **快照原则**：已归档范本的正文以归档时写入的**章节快照**为准；预览、下载、导出等展示路径不得因资源库条款后续修改而改写该快照内容。
- **联动范围**：资源发布或保存触发「同步到关联范本章节」时，**仅更新草稿、已发布范本**的对应章节；**不得**更新已归档范本的章节正文。
- **绑定约束**：新增或编辑资源与范本章节的绑定关系时，**不得**向已归档范本追加或变更绑定（绑定列表选择中须过滤已归档范本）。

### 规则3：审计与异常可追踪

- 创建、更新、删除、同步、发布等关键动作必须记录操作日志。
- 失败场景必须有清晰提示，不得静默失败。
- 关键失败链路需提供可执行回退方案（如重试、兜底、手工处理提示）。

## 细粒度操作逻辑清单

本章用于定义每个细微操作的标准行为，统一产品、开发与测试执行口径。每条按“入口、前置条件、处理步骤、成功结果、失败处理”描述。

### 数据看板

#### 页面进入与数据加载

- 入口：点击左侧菜单“数据看板”。
- 前置条件：无。
- 处理步骤：读取品类、范本、资源数据并计算统计值（**本期不包含**招标文件维度统计）。
- 成功结果：卡片与待办展示最新统计。
- 失败处理：展示空值或默认值，不阻断页面打开。

### 品类管理

#### 点击“同步信息”

- 入口：页面右上“同步信息”按钮。
- 前置条件：当前不在同步中。
- 处理步骤：按钮置为加载态 -> 调用 `POST /api/categories/sync` -> 校验返回结构。
- 成功结果：更新三级树数据、显示成功文案、写入 `category/sync` 审计日志。
- 失败处理：保持原数据不变、恢复按钮状态、提示失败原因。

#### 点击“操作日志”

- 入口：页面右上“操作日志”按钮。
- 前置条件：无。
- 处理步骤：打开日志弹窗 -> 加载 `category/sync` 日志 -> 按时间倒序分页展示。
- 成功结果：展示时间、操作人、操作类型、说明。
- 失败处理：弹窗显示空态“暂无记录”。

#### 树节点展开/折叠

- 入口：点击产业领域或采购分类前的展开图标。
- 前置条件：节点存在子级。
- 处理步骤：切换展开状态集合。
- 成功结果：子节点显示或隐藏。
- 失败处理：无（本地状态切换）。

### 资源管理（一级分组）

#### 资源管理（文本/资格/评标/合同）

##### 点击“新建资源”

- 入口：左侧列表头部“新建”按钮。
- 前置条件：无。
- 处理步骤：打开编辑弹窗并清空表单。
- 成功结果：可录入名称、内容、描述。
- 失败处理：无。

##### 点击“保存草稿”

- 入口：编辑弹窗“保存草稿”按钮。
- 前置条件：名称和内容非空。
- 处理步骤：校验必填 -> 更新或新增资源 -> 资源状态置草稿 -> 更新 `updatedAt` -> 写入版本快照（内容发生变化时）-> 写审计日志。
- 成功结果：弹窗关闭，列表刷新，资源显示“草稿”。
- 失败处理：保留弹窗与输入内容，提示失败。

##### 点击“发布”

- 入口：编辑弹窗“发布”按钮，或详情页“立即发布草稿”按钮。
- 前置条件：名称和内容非空；详情发布需存在草稿内容。
- 处理步骤：发布内容写入正式稿 -> 状态置已发布 -> 清空草稿内容 -> 生成版本快照（最多保留 10 条）-> 同步更新关联非归档范本章节 -> 写审计日志。
- 成功结果：资源状态为已发布，关联范本（非归档）内容更新。
- 失败处理：不覆盖原正式稿，提示失败并保留编辑态。

##### 点击“绑定章节”

- 入口：资源详情右上“绑定章节”。
- 前置条件：当前已选中资源。
- 处理步骤：打开绑定弹窗 -> 加载范本列表 -> 支持搜索、状态筛选、展开章节 -> 多选目标章节。
- 成功结果：可形成候选绑定集。

## 接口与集成需求

### 规范性引用（项目内）

- 《文档中台接口文档》：WPS 文档中台开放能力、鉴权（**WPS-4 / WPS-4-GM** 签名、`access_key`/`secret_key`）、在线编辑/预览 **API 列表**与 **回调 API 列表**（默认 URI 如 `/v1/3rd/file/info`、`/v1/3rd/file/save`、`/v1/3rd/user/info`、`/v1/3rd/file/online`、`/v1/3rd/onnotify` 等，可在应用管理平台配置自定义前缀）。
- 《WPS云文档接入文档(2)》：集团内网 **中台地址**（生产/测试）、**网闸与端口**、DNS、对接申请入口；在线编辑/在线预览开通建议及功能 ID（如 `platform_webedit`、`platform_webview`）。

### 文档中台侧关键接口（目标集成）

- **获取在线编辑链接**：`GET {配置域名}/open/api/edit/v1/files/{file_id}/link`（Query：`type` 等；详见接口文档）；响应 `data.link` 指向 WebOffice 地址。
- **开放接口通用请求头**：`Content-Type: application/json`、`Wps-Docs-Date`、`Wps-Docs-Authorization`（签名算法见《文档中台接口文档》）。
- **业务系统必须实现的回调**（编辑链路）：至少包括 **获取文件元数据**、**上传文件新版本**；按需实现 **获取用户信息**、**文件状态/协作通知**、**回调通知**、重命名、版本历史等（见接口文档「在线编辑—接入流程」及回调列表）。
- **在线预览**（可选）：按文档中台「在线预览」能力与回调配置启用。

### 业务系统衔接（产品逻辑）

- **file_id**：与合作方生成规则一致，须与范本侧文档唯一对应；避免使用易触发异常的非法符号（接口文档建议字母数字）。
- **保存闭环**：用户编辑后由中台触发 **上传文件新版本**，业务系统持久化 DOCX 并 **解析回写范本章节树**；鉴权、存储路径与失败重试策略由实现定义。
- **品类等业务接口**：与中台无关的内部 API 可独立演进；过渡期内具体路由名以实现为准，**不以非文档中台方案的编辑器专属接口为产品目标**。

### 集成要求

- **文档中台** 部署地址、**授权证书** 与 ak/sk 可用；回调 URL 对中台 **可达**（内网策略按《WPS云文档接入文档(2)》开通）。
- 业务系统对回调接口返回结构符合《文档中台接口文档》，错误码遵循《错误码说明》《回调错误码》。
- 文档导入、保存解析、下载链路具备 **异常回退与可观测性**（日志含 `request_id` 等）。

### 接口错误与 HTTP 状态分类

本节描述**接口层**分类方式：与页面 Toast/弹窗文案可以不一致。目标集成以 **文档中台** 与 **业务回调** 的分层错误为准。

#### 约定

- **文档中台开放 API**：响应体含 `code`、`msg`、`request_id` 等（成功时 `code` 一般为 `200`，详见《文档中台接口文档》《错误码说明》）。
- **业务系统回调接口**：响应须符合中台约定；回调异常返回常见字段 `code`、`message`、`details`、`hint`（示例错误码如 `40005 InvalidArgument`，详见《回调错误码》）。
- **业务应用自有接口**（品类同步、内部下载等）：可由实现定义 `{ "error": "<字符串>" }` 等结构，**不作为**文档中台规范性要求。

#### HTTP 状态语义（与本系统集成相关）

| 状态码 | 含义（接口层） | 典型场景 |
|--------|------------------|----------|
| **400** | 请求不合法 | 缺少必填参数、签名参数错误、非法 file_id 等。 |
| **403** | 拒绝执行 | 无权限或策略拒绝（含回调鉴权失败场景，以实现为准）。 |
| **404** | 资源不存在 | 文件未就绪、回调路由未挂载等。 |
| **502** | 网关或上游失败 | 业务系统与中台网络不通、中台侧转换/拉取失败等。 |

#### 文档中台侧错误与排查入口（目标集成）

| 来源 | 说明 |
|------|------|
| 《错误码说明》 | 开放 API `code`/`msg` 全集（含证书、编辑链接、预览等业务）。 |
| 《回调错误码》 | 业务回调返回体中的 `code`/`message`/`details`。 |
| 证书与授权 | 证书到期前提醒回调、后台 licence 查询接口；需在后台关注授权状态（见接口文档「授权管理」）。 |

**说明**：存量工程中若仍存在过渡期的自有 Route Handlers 错误字符串，仅用于改造过程排查；**验收口径以文档中台 + 回调规范为准**。

## 风险与后续建议

### 当前风险

- 本地存储模式不适用于多用户并发场景。
- **文档中台** 可用性、**授权证书**有效期及 **内网/网闸策略** 直接影响在线编辑主链路（参见《WPS云文档接入文档(2)》）。
- 业务回调实现不完整或与中台版本不匹配时，将导致无法加载文件或无法保存新版本。

### 建议规划

- 完成与文档中台的 **测试环境联调** 后再做生产割接；证书与 ak/sk 按运维节奏轮换。
- 在范本变量与章节结构稳定后，落地**基于范本生成招标文件**、变量回填与成品导出。
- 逐步迁移到后端持久化与统一权限体系。
- 增加审批流程与更细粒度审计中心。
- 增强导入解析准确率与模板质量校验规则。
