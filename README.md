# 招标文件范本编制工具平台

政企采购领域的招标文件范本和招标文件编制系统前端应用。

## 技术栈

- **框架**: Next.js 16 (App Router)
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **图标**: Lucide React
- **UI 风格**: 飞书 SaaS 风格

## 功能模块

### 1. 看板管理（主页）
- 展示系统核心业务数据统计
- 待办事项提醒
- 快速操作入口

### 2. 品类管理
- 三级层级结构：产业领域 → 采购分类 → 品类
- 支持新增、编辑、删除、查询操作
- 树形展示，可折叠展开

### 3. 框架管理
- 管理投标文件范本的大纲结构
- 章节树形编辑（支持拖拽排序）
- 框架与品类绑定
- 支持草稿和发布状态

### 4. 文本管理
- 管理固定文本片段
- 文本与框架章节绑定
- 文本修改自动同步到所有绑定位置
- 支持富文本编辑

### 5. 范本管理
- 基于框架自动生成初始范本
- 范本内容编辑
- 变量占位符管理（如 {{招标人}}、{{项目名称}}）
- 支持草稿、已发布、已归档状态

### 6. 招标文件管理
- 基于范本生成招标文件
- 变量替换功能
- 文件预览和下载
- 支持草稿和已完成状态

## 业务流程

1. **编辑维护品类** - 建立三级品类体系
2. **对品类进行框架设计** - 设计章节结构
3. **编辑文本信息并绑定到框架** - 创建可复用的文本片段
4. **系统自动生成初始范本** - 基于框架和绑定文本
5. **编辑初始范本并预留变量** - 完善范本内容
6. **选择范本并生成招标文件** - 填充变量生成最终文件

## 开发指南

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

访问 [http://localhost:3000](http://localhost:3000) 查看应用。

### 构建生产版本

```bash
npm run build
npm start
```

## UI 设计规范

### 色彩体系
- **背景色**: `bg-gray-50` / `bg-gray-100`
- **内容区**: `bg-white`
- **主题色**: `blue-600` (#3370ff - 飞书蓝)
- **文字颜色**:
  - 主标题: `text-slate-900`
  - 正文: `text-slate-700`
  - 辅助文本: `text-slate-500`

### 交互细节
- **阴影**: `shadow-sm` (轻量阴影)
- **边框**: `border-slate-200` (细腻边框)
- **圆角**: `rounded-lg`
- **过渡**: `transition-colors`

### 风险提示色彩
- **高风险**: `rose-500` / `bg-rose-50` (柔和玫瑰红)
- **低风险**: `amber-500` / `bg-amber-50` (琥珀色)

## 数据说明

当前所有数据均为 Mock 数据，存储在各页面组件的 `useState` 中。后续可以：

1. 将 Mock 数据提取到独立的 `lib/mockData.ts` 文件
2. 创建 API 路由 (`app/api/`) 对接真实后端
