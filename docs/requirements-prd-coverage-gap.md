## 3. 缺口清单（建议后续补号或补 PRD）

以下建议在里程碑评审中 **逐项勾选**：补需求行 / 补 PRD / 或标注「文档级不适用」。

**CSV 勾选版**（可导入 Excel/WPS）：`docs/requirements-prd-coverage-gaps.csv`（列含缺口编号、PRD 锚点、描述、建议动作、建议 REQ、状态、备注、负责人、目标关闭日期）。

| 序号 | 缺口描述 | 建议动作 |
|------|----------|----------|
| G1 | **`## 文档信息`** 中印刷版导出规则、WPS 目标集成声明无独立 REQ | 新增 **META-001～002** 或并入交付验收 checklist |
| G2 | **`## 项目背景与目标`** 无映射 | 新增 **BG-001～n** 或将目标 bullet 映射到 ACC/SCOPE（避免重复则写「见 §范围」） |
| G3 | **`## 系统功能结构`**（菜单 IA）无映射 | 新增 **NAV-001**（一级菜单列表）、**NAV-002**（资源四类子菜单） |
| G4 | **资源通用业务流程第 2 步**（加载列表与左右栏）未单独 ID | 接受合并则备注 RM-002；否则新增 **RM-024** 拆分 |
| G5 | **范本「归档 / 解除归档」操作**：PRD 多处规则与术语涉及，**页面层级/细粒度无对应点击条目** | **产品核对**：若已实现 → 补 **TP-0xx** + 细粒度；若本期不做 → PRD 写明「仅规则预留」 |
| G6 | **接口与集成** 下 **`### 规范性引用`** 正文含两篇文档的长 bullet — 已由 REF-001、REF-002 覆盖；若需逐 URI **细化到附录章节号**，增补 REF-005… |
| G7 | **`### 文档中台侧关键接口`** 各 bullet（编辑链接、请求头、回调清单、预览）— 已由 INT-002～005 **单行概括**；若要与 OpenAPI **一对一**，占用 INT-013… |
| G8 | **文档附件 · 接口错误** 中 ####「约定」三下划线 — ERR-001～003 已分三层；**HTTP 表格行** — ERR-004～007；**与 INT-012 重复索引** — 在 INT-012「依赖/备注」注明「详见 ERR」避免重复测试 |
| G9 | **semi-auto / supplement 空号段**（如 RM-024～120）不代表「无需求」，仅为 **未拆分占位** | 发布前决定：**删减号段**或**补全拆条** |
| G10 | **OUT-BID-003～010** 仅占位，与非本期范围 **仅前 2 条**有语义 — 其余空行待远期需求或删除 | 与 **SCOPE-006～009** 去重，避免双维护 |

## 5. 使用方式（勾掉）

1. 打开本清单 **§2**，从左到右确认 **映射 ID** 是否满足本轮验收。  
2. 对 **§3** 每条缺口：在追踪表中 **新建 ID** 或 **更新 PRD** 后关闭缺口。  
3. **§4** 仅在需要 **用例与 PRD 标题 1:1** 时拆分 semi-auto 空行。
