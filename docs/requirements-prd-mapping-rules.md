# PRD ↔ 需求追踪映射规则

本文说明 **`requirements-traceability-semi-auto.csv`**（主表）、**`requirements-traceability-supplement.csv`**（增补表）与 **`PRD_v2.0.md`**（现行规格正文）的对应方式；**`PRD_v1.0.md`** 仍为冻结对照基线，章节标题变更时请同时核对 v2「更新日志」与「PRD_v1 与 v2 对照」章节。映射规则中的 **`§…`** 写法不变。

---

## 1. 「PRD 依据」列写法（§ 约定）

| 写法 | 含义 | 示例 |
|------|------|------|
| `§页面层级-…` | 对应「页面层级功能与操作逻辑」下的 `###` / `####` 标题路径 | `§页面层级-数据看板-综述` |
| `§细粒度-…` | 对应「细粒度操作逻辑清单」下路径 | `§细粒度-品类管理-点击「同步信息」` |
| `§全局规则-…` | 全局规则小节 | `§全局规则-规则2-细化-快照原则` |
| `§接口与集成-…` | 接口与集成章节 | `§接口与集成-文档中台侧关键接口-获取在线编辑链接` |
| `§数据要素-…` | 数据要素总览 | `§数据要素-核心实体-资源` |
| `§文档附件-术语表-…` | 文档体系常见附件 — 术语表中的术语名（用于增补表） | `§文档附件-术语表-WPS 文档中台` |
| `§范围说明-本期范围` / `非本期范围` | 范围边界条目 | `§范围说明-非本期范围-范本生成招标文件` |

**原则**：优先写到「可直接在 Word/Markdown 里搜索到的标题」；同一需求合并多条 PRD  bullet 时，用 **「～」或顿号** 写在同一单元格（见主表中「业务流程-1～2」类写法）。

## 3. 增补表前缀（`supplement` 专用）

与主表 **455 行骨架无关**，使用独立前缀，避免与 DB～ACC 号段混淆；适用于 **数据字典、术语、角色、范围条文、引用文档、错误分层、风险** 等「横切」追踪。

| 前缀 | 含义 | PRD 章节 |
|------|------|----------|
| **DATA** | 核心实体 / 关键字段口径 | `## 数据要素总览` |
| **TERM** | 术语表逐条（对外评审对齐） | `### 术语表与缩写` |
| **ROLE** | 角色与职责 | `## 角色与职责` |
| **SCOPE** | 本期 / 非本期范围条目 | `## 范围说明` |
| **REF** | 规范性引用（外部文档）与集成硬性要求摘要 | `接口与集成` + 附件中涉及文档 |
| **ERR** | 接口错误约定与 HTTP 语义（产品验收口径） | `文档体系常见附件` · 接口错误 |
| **RISK** | 当前风险与建议规划 | `风险与后续建议` |

**合并进主表的方式（可选）**：在 Excel 中将增补表复制为第二个 Sheet，或用 **VLOOKUP/需求 ID** 关联；不要将 **TERM-xxx** 强行改写为 **GL-xxx** 以免破坏全局规则号段语义。

## 4. 类型列取值

| 值 | 适用于 |
|----|--------|
| 功能 | 页面、操作、流程 |
| 规则 | 全局规则、业务约束 |
| 集成 | 文档中台、回调、网络、证书 |
| 非功能 | 性能、稳定、可维护 |
| 验收 | 验收标准条款 |
| 数据 | 实体与字段（增补表 **DATA**） |
| 术语 | 术语表（**TERM**） |
| 范围 | 本期/非本期（**SCOPE**） |
| 角色 | 干系人（**ROLE**） |
| 风险 | 风险与建议（**RISK**） |

## 5. 更新 PRD 时的维护顺序

1. 改 **`PRD_v2.0.md`** 正文（现行规格）；若需保留历史对照，可同步在 **`PRD_v1.0.md`** 文首说明中标注「不得以 v1 替代 v2」，**勿**将 v1 当作最新实施依据。  
2. 在 **`PRD_v2.0.md`**「更新日志」中追加条目（尤其是与代码对齐的实现口径）。  
3. 全局搜索旧 **`§…`** 字符串，更新主表 / 增补表 **PRD 依据** 列。  
4. 若增删章节导致需求合并或拆分：**保留需求 ID**，在「依赖/备注」列写「合并自 xxx / 拆出 xxx」。  
5. 重新导出印刷版 Word（若仍使用 `scripts/md_to_print_docx.py`，默认源文件为 **`PRD_v2.0.md`**）。
