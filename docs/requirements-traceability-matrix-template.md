# 需求追踪矩阵 — 模块分段与表头模板

依据 **`PRD_v2.0.md`** 结构整理（与 v1 冻结稿章节差异见 `PRD_v2.0.md` 文末「PRD_v1.0 与 PRD_v2.0 对照说明」），用于 Excel / 飞书多维表格「对点追踪」。**需求 ID 仅为建议号段**，可按项目习惯微调；勿与 PRD 章节号混用。

---

## 1. 模块与建议 ID 号段

| 前缀 | 模块（与 PRD 对应） | 建议号段 | 建议拆分条数（含异常/边界时的上限） | 说明 |
|------|---------------------|----------|--------------------------------------|------|
| **DB** | 数据看板 | `DB-001`～`DB-020` | 5～12 | 页面进入、指标卡、待办、刷新；可与「非功能-性能」交叉引用 |
| **CM** | 品类管理 | `CM-001`～`CM-040` | 8～18 | 树浏览、搜索、同步、操作日志；每条规则类单独一行便于挂用例 |
| **RM** | 资源管理（含四子模块） | `RM-001`～`RM-120` | 35～80 | 通用：新建/草稿/发布/绑定/解绑/删除/日志；四子模块差异小可「子类型」列区分 |
| **RM-TXT** | （可选）文本管理 | `RM-TXT-001`～ | 8～15 | 仅当要对四模块分开统计时使用 |
| **RM-QLF** | （可选）资格条件 | `RM-QLF-001`～ | 8～15 | 同上 |
| **RM-PB** | （可选）评标办法 | `RM-PB-001`～ | 8～15 | 同上 |
| **RM-HT** | （可选）合同条款 | `RM-HT-001`～ | 8～15 | 同上 |
| **TP** | 范本管理 | `TP-001`～`TP-080` | 25～55 | 新建、导入、筛选、编辑、保存、预览、下载、发布、删除、日志、归档/解除归档（若验收） |
| **WE** | 在线编辑（WPS 文档中台） | `WE-001`～`WE-060` | 18～45 | 编辑链接、回调 file/info & save、JSSDK、插入资源/变量、失败兜底；可与 **INT** 联动 |
| **PU** | 公共交互规范 | `PU-001`～`PU-025` | 8～18 | 列表分页、弹窗、加载/成功/失败反馈 |
| **GL** | 全局规则 | `GL-001`～`GL-025` | 10～22 | 状态统一、归档快照、绑定约束、审计；每条规则至少 1 条负例用例 |
| **INT** | 接口与集成（文档中台） | `INT-001`～`INT-040` | 12～35 | 鉴权、回调清单、网闸、证书；与《文档中台接口文档》《WPS云文档接入文档》对条 |
| **NFR** | 非功能需求 | `NFR-001`～`NFR-020` | 6～15 | 性能、稳定、可维护；尽量量化或写「观测方式」 |
| **ACC** | 验收标准（交叉） | `ACC-001`～`ACC-015` | 5～12 | 功能/交互/一致性总条款拆行，便于里程碑抽检 |

## 2. 追踪表列定义（复制到表格首行）

| 列名 | 必填 | 说明 |
|------|------|------|
| 需求 ID | 是 | 上表前缀 + 序号 |
| 模块 | 是 | 与前缀一致，便于筛选 |
| 需求名称 | 是 | ≤20 字为宜 |
| 类型 | 否 | 功能 / 规则 / 集成 / 非功能 / 验收 |
| PRD 依据 | 是 | 章节标题或「细粒度」小节名，可写行号 |
| 描述 | 是 | 用户视角一句话或验收视角 |
| 前置条件 | 否 | 数据、权限、网络（如中台可达） |
| 验收标准（可测） | 是 | 可观察结果；接口类写「返回字段/状态码」 |
| 优先级 | 否 | P0 / P1 / P2 |
| 状态 | 否 | 草稿 / 已评审 / 开发中 / 已测 / 挂起 |
| 测试用例 ID | 否 | 与用例库关联 |
| 依赖 / 备注 | 否 | 如依赖 INT-003、仅测试环境 |
| **变更标注** | 否 | 与 PRD 某版对齐时填写 **`更新`** / **`新增`**；已废止、仅作对照的口径填 **`删除`**（或见表末 **`OBS-*`** 行）。Excel / 飞书可对末列做条件格式：**更新、新增 → 红色字体**，**删除 → 灰色或删除线**。浏览器可直接打开带样式导出页：`docs/requirements-traceability-skeleton-highlighted.html`（由 `scripts/annotate_traceability_skeleton.py` 从 CSV 生成）。 |

## 4. 与 PRD 细粒度清单的映射提示（减轻拆分工作量）

| PRD 区块 | 建议一行需求 ID 粒度 |
|----------|----------------------|
| 「细粒度操作逻辑清单」下每个 `####` 标题 | 优先 **1 标题 = 1 行**；复杂操作拆成「主成功 / 失败回滚」两行 |
| 「全局规则」规则 2 细化 bullet | **每条 bullet = 1 行 GL**，便于归档/绑定专项回归 |
| 「在线编辑」插入资源/变量 | **WE**，并与 RM/TP 交叉备注「绑定补写」 |

## 5. 映射规则与增补 CSV（数据要素 / 术语 / 范围等）

- **映射规则全文**：`docs/requirements-prd-mapping-rules.md`（§ 写法、主表前缀、增补前缀 **DATA / TERM / ROLE / SCOPE / REF / ERR / RISK**、类型列、PRD 变更维护顺序）。

- **PRD 目录 ↔ 追踪表覆盖与缺口清单**：`docs/requirements-prd-coverage-gap.md` — 按 PRD `##`/`###` 勾对 semi-auto/supplement 中的 ID，并列出未映射章节与建议补号（META/NAV/范本归档等）。

## 6. 版本与维护

- 表结构变更时：**需求 ID 不改**，仅改「状态/用例」列，保证跨版本追踪。
- PRD 升级时：在「PRD 依据」列更新章节引用，必要时新建行并注明「替代 DB-003」等；同步检查 **增补表** 中 **TERM / DATA** 是否与术语表、数据要素章节一致。
