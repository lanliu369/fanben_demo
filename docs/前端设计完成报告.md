# 招标文件范本编制工具平台 - 前端设计完成

## 项目概述

已完成招标文件范本编制工具平台的前端设计和实现，包含 6 个核心功能模块，采用飞书 SaaS 风格的 UI 设计。

## 已完成的功能模块

### ✅ 1. 看板管理页面 (DashboardPage)
- 4 个统计卡片：品类总数、框架总数、范本总数、招标文件数
- 待办事项列表（支持警告和信息两种类型）
- 快速操作入口（新建品类、设计框架、编制范本）

### ✅ 2. 品类管理页面 (CategoryPage)
- 三级层级结构：产业领域 → 采购分类 → 品类
- 树形展示，支持折叠/展开
- 完整的 CRUD 操作（新增、编辑、删除）
- 搜索功能
- 模态框表单编辑

### ✅ 3. 框架管理页面 (FrameworkPage)
- 左侧框架列表，右侧章节编辑器
- 章节树形结构展示
- 支持添加、编辑、删除章节
- 支持添加子章节（多级嵌套）
- 框架状态管理（草稿/已发布）
- 基于品类创建框架

### ✅ 4. 文本管理页面 (TextPage)
- 文本片段列表展示
- 文本内容编辑（支持多行文本）
- 文本绑定到框架章节
- 显示绑定关系列表
- 支持一个文本绑定到多个章节
- 搜索功能

### ✅ 5. 范本管理页面 (TemplatePage)
- 基于框架自动生成范本
- 范本章节内容编辑
- 变量管理（识别 {{变量名}} 格式）
- 范本预览功能
- 范本状态管理（草稿/已发布/已归档）
- 发布和归档操作

### ✅ 6. 招标文件管理页面 (BidDocumentPage)
- 基于范本创建招标文件
- 变量值填充表单
- 自动识别范本中的变量
- 文件状态管理（草稿/已完成）
- 文件定稿功能
- 下载功能（占位）

## 技术实现

### 核心技术栈
- **Next.js 16** - App Router 架构
- **TypeScript** - 完整的类型定义
- **Tailwind CSS** - 飞书风格的 UI 设计
- **Lucide React** - 图标库

### 组件结构
```
src/
├── app/
│   ├── layout.tsx          # 根布局（中文、灰色背景）
│   ├── page.tsx            # 主页入口
│   └── globals.css         # 全局样式
├── components/
│   ├── MainLayout.tsx      # 主布局 + Tab 导航
│   └── pages/              # 6 个功能页面
│       ├── DashboardPage.tsx
│       ├── CategoryPage.tsx
│       ├── FrameworkPage.tsx
│       ├── TextPage.tsx
│       ├── TemplatePage.tsx
│       └── BidDocumentPage.tsx
└── types/
    └── index.ts            # 完整的 TypeScript 类型定义
```

### 数据类型定义
- `Industry` - 产业领域
- `ProcurementCategory` - 采购分类
- `Category` - 品类
- `Framework` - 框架
- `Chapter` - 章节
- `TextFragment` - 文本片段
- `TextBinding` - 文本绑定关系
- `Template` - 范本
- `TemplateSection` - 范本章节
- `TemplateVariable` - 范本变量
- `BidDocument` - 招标文件
- `VariableValue` - 变量值

## UI 设计特点

### 飞书 SaaS 风格
1. **色彩克制**
   - 背景：灰白色 (bg-gray-50)
   - 内容区：纯白 (bg-white)
   - 主题色：飞书蓝 (blue-600, #3370ff)
   - 文字：slate 系列（900/700/500）

2. **交互细腻**
   - 轻量阴影 (shadow-sm)
   - 细腻边框 (border-slate-200)
   - 圆角 (rounded-lg)
   - 平滑过渡 (transition-colors)

3. **信息层级清晰**
   - 卡片式布局
   - 合理的间距和留白
   - 清晰的视觉层次

## 业务流程完整性

系统完整实现了 6 步业务流程：

1. ✅ 编辑维护品类
2. ✅ 对品类进行框架设计
3. ✅ 编辑文本信息并绑定到框架
4. ✅ 系统自动生成初始范本
5. ✅ 编辑初始范本并预留变量
6. ✅ 选择范本并生成招标文件
