# 投标文件范本编制工具平台功能说明书

## 一、系统概述

本系统是一款投标文件范本编制工具平台，核心目标是通过模块化编辑方式，实现各类投标文件范本的规范化、高效化管理，支持从品类维护、框架设计到最终招标文件生成的全流程业务操作，降低人工编制成本，确保范本及招标文件的标准化、一致性。系统整体采用tab页导航设计，各功能模块独立分工、协同联动，贴合用户实际业务操作习惯。

## 二、核心页面功能模块

系统包含6个核心tab页，各页面功能独立且关联，具体功能如下：

### 2.1 看板管理（主页）

作为系统默认主页，主要用于展示系统核心业务数据及操作入口，方便用户快速了解系统运行状态、待办事项（如未完成的范本编制、待审核的框架等），并可快速跳转至其他各功能页面，提升操作便捷性。

### 2.2 品类管理页面

核心功能是对投标文件范本的品类进行全生命周期管理，明确品类的层级关系，为后续框架设计、范本编制提供分类依据。

品类属性层级：采用三级层级结构，即「产业领域」→「采购分类」→「品类」，三者为从属关系，一个产业领域可包含多个采购分类，一个采购分类可包含多个品类，层级清晰可追溯。

核心操作：支持品类的新增、编辑、删除、查询，可维护各层级的名称、描述等基础信息，确保品类分类符合实际业务需求。

### 2.3 框架管理页面

核心功能是管理投标文件范本的大纲（目录章节结构），为范本编制提供统一的结构规范，确保同品类范本的章节一致性。

核心操作：

- 框架新增：可创建新的框架，绑定对应的品类（明确该框架属于哪个产业领域-采购分类-品类），设置框架名称、描述等基础属性。

- 框架编辑：可修改框架的章节结构（如新增章节、删除章节、调整章节顺序、修改章节名称），编辑完成后保存，同步影响后续基于该框架生成的范本。

- 框架关联：每个框架必须绑定唯一品类，确保框架与品类一一对应，避免混乱。

### 2.4 文本管理页面

核心功能是管理各类固定文本片段，通过将文本与框架章节绑定，实现范本内容的自动更新，减少重复编辑工作，确保文本内容的统一性。

核心操作：

- 文本编辑：支持固定文本的新增、编辑、删除、查询，可录入文本内容、设置文本名称、描述等信息。

- 文本绑定：将编辑好的固定文本，关联到指定框架的具体章节，一个章节可绑定多个固定文本，一个文本可绑定到多个框架的对应章节；当文本内容修改后，所有绑定该文本的框架章节内容将自动同步更新。

### 2.5 范本管理页面

核心功能是基于已设计好框架的品类，进行投标文件范本的编制、保存与管理，是连接框架、文本与招标文件的核心环节。

核心操作：

- 范本生成：选择已维护好框架的品类，系统自动根据该品类的框架结构，以及绑定在各章节的固定文本，生成初始范本（初始范本包含框架章节及绑定的固定文本内容）。

- 范本编辑：用户可对初始范本进行编辑，补全缺失的信息，同时预留必要的变量信息（如招标编号、招标人、招标时间、项目名称等），变量信息需明确标识，方便后续生成招标文件时替换。

- 范本保存：编辑完成的范本可保存至系统，按品类、框架进行分类归档，支持查询、修改、删除等操作。

### 2.6 招标文件管理页面

核心功能是基于已编制完成的范本，通过替换变量信息，生成符合实际需求的正式招标文件，完成整个业务闭环。

核心操作：

- 范本引用：用户从系统中选择已保存的范本，作为生成招标文件的基础。

- 变量替换：系统自动识别范本中预留的变量信息，用户编辑录入实际的变量内容（如填写具体的招标编号、招标人名称、招标时间等），替换完成后生成正式的招标文件。

- 招标文件管理：生成的招标文件可保存、下载、查询，支持后续修改（修改时可重新编辑变量信息或引用的范本内容）。

## 三、系统业务使用流程

系统业务流程遵循“从基础设置到最终输出”的逻辑，共分为6个步骤，各步骤依次衔接，确保业务流程顺畅、规范，具体如下：

### 步骤1：编辑维护品类

用户进入品类管理页面，按照「产业领域」→「采购分类」→「品类」的三级结构，新增、编辑各类品类信息，明确品类的层级关系和基础属性，完成品类的维护，为后续框架设计提供分类依据。

### 步骤2：对品类进行框架设计

用户进入框架管理页面，为已维护好的品类创建对应的框架，设计框架的目录章节结构（新增、调整章节），设置框架的基础信息并绑定对应品类，完成框架的设计与保存。

### 步骤3：编辑文本信息并绑定到框架上

用户进入文本管理页面，编辑各类固定文本片段，完成文本的新增、修改后，将文本绑定到对应框架的具体章节，实现文本与框架章节的关联，为后续范本自动生成做准备。

### 步骤4：系统自动生成初始范本

用户进入范本管理页面，选择已完成框架设计的品类，系统自动读取该品类对应的框架结构，以及绑定在各章节的固定文本，生成初始范本，初始范本包含完整的章节结构和绑定的固定文本内容。

### 步骤5：编辑初始范本并预留变量

用户对系统生成的初始范本进行编辑，补全范本中缺失的信息，同时根据实际招标需求，在范本中预留必要的变量信息（如招标编号、招标人、招标时间等），并对变量进行明确标识，编辑完成后保存范本。

### 步骤6：选择范本并生成招标文件

用户进入招标文件管理页面，选择已编辑完成的范本，系统自动识别范本中的预留变量，用户录入实际的变量内容，完成变量替换后，系统生成正式的招标文件，用户可对招标文件进行保存、下载或进一步修改。

## 四、核心业务规则

- 品类层级规则：必须先维护产业领域，再维护该产业领域下的采购分类，最后维护采购分类下的品类，不可跨层级创建。

- 框架绑定规则：每个框架必须绑定唯一品类，一个品类可对应多个框架（适用于同一品类不同场景的范本需求）。

- 文本绑定规则：文本可绑定到多个框架的对应章节，文本修改后，所有绑定该文本的框架章节内容自动同步更新。

- 范本生成规则：只有已完成框架设计且绑定了文本的品类，才能生成初始范本；初始范本生成后，可多次编辑修改。

- 变量规则：范本中预留的变量需明确标识，生成招标文件时，所有变量必须替换为实际内容，方可完成招标文件生成。
