# 招标文件范本系统：交互方案与功能设计（前后端联调版）

## 1. 文档目标与范围

本文档用于统一前端、后端、联调、测试的实现口径，覆盖当前仓库已落地能力与联调边界，重点包括：

- 业务主链路与页面交互
- 核心数据模型与字段关系
- OnlyOffice 编辑、插入、导出、回调链路
- API 合同（当前 Next.js API 实现）
- 已知边界与后续服务化建议

适用对象：前端开发、后端开发、测试工程师、技术负责人。

---

## 2. 系统概览

### 2.1 业务定位

系统面向政企采购招标文件编制，提供“资源沉淀 -> 范本生成/编辑 -> 招标文件生成”的闭环。

### 2.2 技术形态

- 前端：Next.js App Router + TypeScript + Tailwind CSS + Lucide React
- 数据：本地 `localStorage` 持久化（Mock 链路）
- 文档编辑：OnlyOffice Document Server（Docker 部署）
- BFF：Next.js API Routes
- 文档文件：`public/documents`（docx）

### 2.3 页面构成

- 数据看板
- 品类管理
- 范本管理（含预览、导入、OnlyOffice 编辑）
- 资源管理（文本/资格条件/评标办法/合同条款）
- 招标文件管理

---

## 3. 领域模型与关系

### 3.1 核心实体

- `Industry` -> `ProcurementCategory` -> `Category`
- `Template`
  - `TemplateSection`（支持层级）
  - `TemplateVariable`
- `TextFragment`
  - `TextBinding`
- `BidDocument`

### 3.2 关键关联规则

- 资源章节绑定：`TextBinding.templateId + templateSectionId`
- 兼容旧绑定：`frameworkId + chapterId`（读兼容）
- 招标文件引用范本并填充变量
- 看板统计基于实时关联关系计算

## 4. 模块功能设计（按页面）

### 4.1 品类管理

- 维护三级结构（产业领域/采购分类/品类）
- 顶部主操作改为“同步信息”
  - 前端触发 `POST /api/categories/sync`
  - 当前返回 mock 数据并回填页面
  - 展示同步中状态与结果提示文案

### 4.2 资源管理

- 四类资源统一管理：文本、资格条件、评标办法、合同条款
- 支持新增、编辑、状态切换（草稿/已发布）
- 支持“绑定范本章节”：
  - 范本行默认收起
  - 提供关键词筛选 + 状态筛选
  - 支持多范本、多章节勾选绑定

### 4.3 范本管理

- 范本新增、筛选、状态管理、导入/导出、编辑
- 预览弹窗支持章节内容正确渲染：
  - HTML 内容按 HTML 渲染
  - 被转义 HTML 自动解码后渲染
  - 纯文本自动转段落并保留换行
  - 递归展示子章节（不止顶层）

### 4.4 导入预览（文档导入弹窗）

- 支持 `.doc/.docx/.pdf` 选择或拖拽
- 提供“渲染预览 / HTML 源码”双视图
- 解析后生成章节树并导入为范本

### 4.5 范本编辑（OnlyOffice 右侧系统功能区）

已实现能力：

- 资源数据与当前范本联动（非硬编码）
- 未绑定资源以资源池形式兜底展示
- 资源分类页签：文本/资格/评标/合同
- 变量库页签：新增变量、插入变量、回写范本变量
- 章节选择器：作为插入后自动绑定目标章节
- 关键词筛选：按标题/摘要/章节标签/状态过滤
- 操作统一 icon：插入（`+`）与复制

插入与绑定策略：

1. 优先直接插入编辑器（多 API + 重试）
2. 失败走插入队列（插件或前端消费）
3. 插入动作自动写入 `TextBinding`（按所选章节）
4. 若仍失败，提示手动粘贴（保底）

安全约束：

- 已移除“插入失败后重建整份文档”的破坏性兜底
- 目标是“失败可回退，但不改坏原文档其他内容”

### 4.6 招标文件管理

- 基于范本生成招标文件
- 变量填充与完成度推进
- 状态管理：草稿/完成/归档

### 4.7 数据看板

- 指标基于真实本地数据关系计算
- 已移除“待审核范本”待办项（当前系统无审核流）

## 5. 关键交互流程

### 5.1 资源复用主链路

1. 在资源管理维护资源内容
2. 在资源管理绑定范本章节
3. 打开范本编辑器自动看到关联资源
4. 点击插入/复制
5. 插入时自动补写绑定（未绑定时）

### 5.2 范本编辑保存链路

1. 点击“保存并返回”
2. 优先 `downloadAs('html')`
3. 失败快速回退 `/api/documents/[id]/export-html`
4. 解析并合并回 `TemplateSection`
5. 更新范本元数据（含变量、更新时间）

### 5.3 导出链路（样式清洗）

导出前清洗内容：

- 去除资源插入高亮块样式（背景/边框/颜色）
- 兼容历史高亮块结构识别
- 去除分隔线文本（`────`、`----`）

## 6. API 设计（当前实现）

### 6.1 文档编辑域（OnlyOffice）

- `GET /api/documents/[id]`：读取 docx
- `POST /api/documents/[id]`：覆盖上传 docx（base64）
- `GET /api/documents/[id]/editor-config`：返回 editorConfig（token、callback、plugins）
- `POST /api/documents/[id]/callback`：OnlyOffice 回调保存
- `PUT /api/documents/[id]/heading`：兜底生成标题文档
- `POST /api/documents/[id]/export-html`：服务端 converter 导出 html
- `GET /api/onlyoffice-proxy-file`：代理拉取 DS 文件

### 6.2 插入队列域

- `POST /api/documents/[id]/insert-queue`：插入请求入队
- `GET /api/documents/[id]/insert-queue`：查询队列长度
- `GET /api/documents/[id]/insert-queue?take=1`：消费一条
- `OPTIONS /api/documents/[id]/insert-queue`：CORS 预检

### 6.3 插件清单与静态资源

- `GET /api/onlyoffice-plugin/resource-insert?docId=...`：返回插件 manifest
- 插件静态目录：`public/onlyoffice-plugins/resource-insert/`
  - `index.html`
  - `loader.js`
  - `plugin.js`
  - `config.json`
  - `icon.svg`

### 6.4 品类同步预留接口

- `POST /api/categories/sync`
  - 当前返回 mock industries 数据
  - 作为后续外部平台同步接入口

## 8. OnlyOffice 集成说明与边界

### 8.1 关键环境变量

- `NEXT_PUBLIC_ONLYOFFICE_DOCUMENT_SERVER_URL`
- `ONLYOFFICE_PUBLIC_BASE_URL`
- `ONLYOFFICE_INTERNAL_BASE_URL`
- `ONLYOFFICE_CALLBACK_BASE_URL`
- `ONLYOFFICE_JWT_SECRET`

### 8.2 当前稳定性结论

- 文档打开、编辑、回调、保存、导出链路可用
- 直接插入受 DS 版本与 iframe 环境影响，存在差异
- 当前采用“重试 + 队列 + 手动粘贴提示”的多层兜底

### 8.3 已闭环问题（摘要）

- `errorCode=-4` 下载失败：地址可达性与回退链路优化
- 插件资源 404：补齐 `config.json`、使用可访问图标路径
- `spellcheck` 弃用告警：迁移至 `customization.features.spellcheck`
- 导出残留高亮/分隔线：导出清洗逻辑处理
- 范本预览显示 HTML 源码：已修复为 HTML 正确渲染

## 9. 联调测试清单（建议）

### 9.1 资源与范本编辑

- 资源是否按“当前范本关联 + 资源池”正确展示
- 资源筛选（关键词/状态）是否生效
- 插入后是否自动生成章节绑定
- 插入失败是否仅提示且不影响其他文档内容

### 9.2 范本预览与导入

- 范本预览是否正确渲染 HTML（不再显示标签源码）
- 纯文本是否保留换行展示
- 子章节是否递归显示
- 导入弹窗“渲染预览/HTML 源码”切换是否正常

### 9.3 保存、导出与插件

- `downloadAs` 失败时是否回退 `/export-html`
- 导出文档是否去除高亮与分隔线
- `editor-config` plugins 配置是否正确
- `insert-queue` 的 POST/GET/take 是否可用
- 插件静态文件是否全部 200

### 9.4 品类同步与看板

- “同步信息”按钮状态与回填数据是否正确
- 看板待办中不应出现“待审核范本”

## 10. 后端服务化建议

建议按以下顺序迁移：

1. 资源、范本、招标文件 CRUD 服务化
2. 绑定关系与统计计算服务化
