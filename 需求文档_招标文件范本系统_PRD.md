# 招标文件范本系统需求文档（PRD）

## 1. 文档信息

- 文档名称：招标文件范本系统需求文档（PRD）
- 版本：v1.0（历史整理稿；**现行规格请以仓库根目录 `PRD_v2.0.md` 为准**）
- 适用对象：产品经理、前端开发、后端开发、测试工程师、项目管理
- 关联文档：
  - **`PRD_v2.0.md`**（现行）、`PRD_v1.0.md`（冻结对照）

  - `系统框架与交互全景说明.md`

---

## 2. 项目背景

招标文件编制存在条款复用难、范本版本维护难、人工复制粘贴低效且易错的问题。系统目标是通过“资源管理 + 范本管理 + 在线编辑 + 招标文件生成”构建标准化编制流程，提升复用率、编制效率与交付质量。

---

## 3. 产品目标

### 3.1 业务目标

- 建立招标条款标准资源库，实现跨范本复用
- 建立范本标准化生产机制，支撑快速生成招标文件
- 降低人工编辑风险，确保内容结构与变量可控

### 3.2 系统目标

- 支持完整业务闭环：资源 -> 范本 -> 招标文件
- 提供 OnlyOffice 在线编辑能力与保存/导出链路
- 提供可联调 API 契约，支持后续服务化迁移

---

## 4. 用户角色与使用场景

### 4.1 角色定义

- 业务运营：维护品类、维护资源、维护范本
- 编制人员：编辑范本、插入资源、设置变量
- 项目执行人员：基于范本生成招标文件并填充变量
- 测试与联调人员：验证接口链路和页面交互一致性

### 4.2 典型场景

- 维护某行业资审条款，批量复用到多个范本章节
- 在范本编辑时快速插入资源并自动关联章节
- 导出文档时自动去除编辑辅助样式，输出干净正文

---

## 5. 需求范围

### 5.1 本期范围（In Scope）

- 品类管理（三级结构维护 + 同步信息入口）
- 资源管理（文本、资格条件、评标办法、合同条款）
- 范本管理（创建、筛选、导入、预览、编辑、导出）
- OnlyOffice 在线编辑与右侧系统功能区
- 招标文件管理（生成、变量填写、状态推进）
- 数据看板（真实数据统计）

### 5.2 非本期范围（Out of Scope）

- 多组织权限与审批流
- 多人实时协同编辑冲突解决
- 完整审计中心与操作回放
- 生产级数据库与分布式存储方案

---

## 6. 功能需求明细

## 6.1 品类管理

### FR-01 三级品类维护

- 系统应支持维护产业领域 -> 采购分类 -> 品类三级结构
- 不允许跨级创建，必须按层级新增

### FR-02 同步信息按钮

- 页面应提供“同步信息”按钮
- 点击后调用 `POST /api/categories/sync`
- 同步中显示加载状态；完成后显示成功/失败提示
- 当前阶段允许返回 mock 数据并回填页面

验收标准：
- 按钮文案在同步时变为“同步中...”
- 同步成功后列表数据发生可见变化或提示成功消息

## 6.2 资源管理

### FR-03 四类资源统一管理

- 系统应支持文本、资格条件、评标办法、合同条款四类资源
- 每类资源应支持新增、编辑、删除、状态切换（草稿/已发布）

### FR-04 绑定范本章节

- 支持将资源绑定到范本章节
- 范本列表默认折叠
- 支持关键词筛选、状态筛选
- 支持多范本多章节勾选绑定

验收标准：
- 绑定后在范本编辑右侧可看到对应资源
- 未绑定资源仍可在资源池展示

## 6.3 范本管理

### FR-05 范本生命周期管理

- 支持范本新增、筛选、状态管理、导入、导出、编辑入口

### FR-06 预览渲染正确性

- 预览弹窗中章节内容应按真实内容渲染：
  - HTML 内容渲染为富文本
  - 转义 HTML 自动解码后渲染
  - 纯文本自动转段落并保留换行
  - 子章节递归展示

验收标准：
- 不应出现 HTML 标签源码直接显示的问题
- 父子章节结构在预览中可完整展开查看

## 6.4 OnlyOffice 范本编辑

### FR-07 右侧系统功能区

- 展示与当前范本关联资源
- 资源不足时展示资源池兜底数据
- 支持分类页签（文本/资格/评标/合同/变量库）
- 资源项支持关键词筛选（标题、摘要、章节标签、状态）
- 资源操作统一 icon（插入、复制）

### FR-08 变量库

- 支持新增变量
- 支持点击变量插入当前光标位置
- 变量应写回范本变量列表并参与后续招标文件生成

### FR-09 资源插入可靠性

- 插入应优先走编辑器直接插入（多 API + 重试）
- 直接插入失败应走插入队列兜底
- 仍失败时应提示用户手动粘贴
- 插入动作应自动补写章节绑定（基于当前所选目标章节）
- 禁止采用“失败后重建整份文档”的破坏性方案

验收标准：
- 插入失败不会破坏文档其他内容
- 插入失败时有明确提示且可继续手工完成操作

### FR-10 保存与回调

- 支持保存并返回
- 优先从编辑器导出 HTML；失败回退服务端 `/export-html`
- 将解析结果回写 `TemplateSection`，并更新 `updatedAt`

## 6.5 导出需求

### FR-11 导出清洗

- 导出文档前应清理编辑辅助样式
- 必须去除：
  - 资源高亮块样式（背景、边框、颜色等）
  - 分隔线文本（`────`、`----`）

验收标准：
- 导出文档为可直接交付的干净正文
- 不残留编辑器辅助视觉标识

## 6.6 招标文件管理

### FR-12 招标文件生成与变量填充

- 支持基于范本生成招标文件
- 支持变量填写、完成度推进、状态管理（草稿/完成/归档）

## 6.7 数据看板

### FR-13 统计口径

- 看板指标需基于真实数据动态计算
- 不展示“待审核范本”待办项（当前无审核流）

---

## 7. 非功能需求

### NFR-01 稳定性

- 编辑、保存、导出主链路可连续使用
- 插入失败应有兜底策略，不影响其他功能继续操作

### NFR-02 性能

- 主要列表操作（筛选、切换）应在可感知流畅范围内
- 同步和导出操作应给出明确加载反馈

### NFR-03 可维护性

- 接口语义清晰，可平滑迁移至后端服务
- 关键链路具备可观测提示（错误提示、状态文案）

---

## 8. 接口需求（当前联调口径）

- 文档编辑域：
  - `GET /api/documents/[id]`
  - `POST /api/documents/[id]`
  - `GET /api/documents/[id]/editor-config`
  - `POST /api/documents/[id]/callback`
  - `PUT /api/documents/[id]/heading`
  - `POST /api/documents/[id]/export-html`
  - `GET /api/onlyoffice-proxy-file`
- 插入队列：
  - `POST /api/documents/[id]/insert-queue`
  - `GET /api/documents/[id]/insert-queue`
  - `GET /api/documents/[id]/insert-queue?take=1`
- 插件清单：
  - `GET /api/onlyoffice-plugin/resource-insert?docId=...`
- 品类同步：
  - `POST /api/categories/sync`

---

## 9. 验收标准（UAT）

- 资源可维护、可绑定、可在编辑器可见
- 编辑器资源可插入/复制，插入失败有可执行兜底
- 插入后自动绑定逻辑生效
- 范本预览不再出现 HTML 源码显示问题
- 导出文档无高亮样式与分隔线残留
- 品类同步按钮状态与提示正确
- 看板不出现“待审核范本”

---

## 10. 风险与依赖

- OnlyOffice 不同版本 API 行为可能存在差异
- Docker 内外网地址配置错误会导致编辑器下载失败
- 当前本地存储模式不适用于生产多用户并发

---

## 11. 里程碑建议

- M1：当前联调版稳定（功能齐套 + 关键问题闭环）
- M2：后端 CRUD 服务化替换本地存储
- M3：审计、权限、版本化、监控补齐

---

## 12. 后续扩展建议

- 增加审批/审核流（如“待审核范本”重新引入）
- 增加资源版本快照与回滚机制
- 增加协同编辑冲突处理策略
- 增加操作日志与全链路观测能力
